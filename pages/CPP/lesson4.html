C++ Object-Oriented Programming

OOP is a programming paradigm based on the concept of "objects" that contain data and code. The main principles of OOP are:

  • Encapsulation - Bundling data and methods that work on that data within one unit
  • Inheritance - Creating new classes that are built upon existing classes
  • Polymorphism - Ability to present the same interface for different underlying forms (data types or classes)
  • Abstraction - Hiding complex implementation details and showing only necessary features

Example:

// Basic class example class Car { public: string brand; string model; int year; void displayInfo() { cout << year << " " << brand << " " << model << endl; } };

C++ Classes/Objects

A class is a user-defined data type that serves as a blueprint for creating objects. Objects are instances of a class.

Example:

class Rectangle { private: int width; int height; public: void setDimensions(int w, int h) { width = w; height = h; } int getArea() { return width * height; } }; int main() { Rectangle rect; // Create an object rect.setDimensions(5, 3); // Set values cout << "Area: " << rect.getArea() << endl; // Use object return 0; }

C++ Class Methods

Class methods are functions that belong to a class. They can be defined inside or outside the class definition.

Inside Class Definition:

class Calculator { public: // Method defined inside class int add(int a, int b) { return a + b; } };

Outside Class Definition:

class Calculator { public: // Method declared inside class int multiply(int a, int b); }; // Method defined outside class using scope resolution :: int Calculator::multiply(int a, int b) { return a * b; }

C++ Constructors

Basic Constructor

A constructor is a special member function that is automatically called when an object is created. It has the same name as the class and is used to initialize object attributes.

Default Constructor:

class Student { private: string name; int age; public: // Default constructor Student() { name = "Unknown"; age = 0; } };

Parameterized Constructor:

class Rectangle { private: double width; double height; public: // Constructor with parameters Rectangle(double w, double h) { width = w; height = h; } };

Constructor Overloading

Constructor overloading allows a class to have multiple constructors with different parameters.

Multiple Constructors:

class Car { private: string brand; string model; int year; public: // Default constructor Car() { brand = "Unknown"; model = "Unknown"; year = 2024; } // Constructor with brand only Car(string b) { brand = b; model = "Unknown"; year = 2024; } // Constructor with all parameters Car(string b, string m, int y) { brand = b; model = m; year = y; } };

C++ Access Specifiers

Access specifiers define how class members can be accessed:

  • public: Members are accessible from outside the class
  • private: Members can only be accessed within the class
  • protected: Members can be accessed in inherited classes

Example:

class BankAccount { private: double balance; // Only accessible within class protected: string accountType; // Accessible in derived classes public: void deposit(double amount) { // Accessible everywhere balance += amount; } double getBalance() { return balance; } };

C++ Encapsulation

Encapsulation is the bundling of data and the methods that operate on that data within a single unit (class), hiding internal details.

Example:

class Employee { private: string name; double salary; public: void setName(string n) { name = n; } string getName() { return name; } void setSalary(double s) { if (s > 0) { // Data validation salary = s; } } double getSalary() { return salary; } };

C++ Inheritance

Basic Inheritance

Inheritance allows a class to inherit attributes and methods from another class. The class being inherited from is called the base class (or parent class), and the class that inherits is called the derived class (or child class).

Example:

class Animal { // Base class public: string species; void eat() { cout << "This animal is eating" << endl; } }; class Dog : public Animal { // Derived class public: string breed; void bark() { cout << "Woof!" << endl; } }; int main() { Dog myDog; myDog.species = "Canine"; // Inherited from Animal myDog.breed = "Labrador"; // Dog's own attribute myDog.eat(); // Inherited method myDog.bark(); // Dog's own method return 0; }

Multilevel Inheritance

In multilevel inheritance, a derived class inherits from another derived class. This creates a hierarchy of classes.

Example:

class Animal { // Base class public: void eat() { cout << "Eating..." << endl; } }; class Mammal : public Animal { // First derived class public: void breathe() { cout << "Breathing..." << endl; } }; class Dog : public Mammal { // Second derived class public: void bark() { cout << "Barking..." << endl; } }; int main() { Dog myDog; myDog.eat(); // From Animal class myDog.breathe(); // From Mammal class myDog.bark(); // From Dog class return 0; }

Multiple Inheritance

Multiple inheritance allows a class to inherit from more than one base class. This can be useful but needs to be used carefully to avoid ambiguity.

Example:

class Car { public: void drive() { cout << "Driving..." << endl; } }; class Boat { public: void sail() { cout << "Sailing..." << endl; } }; class AmphibiousVehicle : public Car, public Boat { public: void travel() { cout << "Traveling on land and water!" << endl; drive(); // From Car sail(); // From Boat } }; int main() { AmphibiousVehicle vehicle; vehicle.drive(); // From Car vehicle.sail(); // From Boat vehicle.travel(); // Own method return 0; }

Inheritance Access Specifiers

The access specifier used in inheritance affects how the base class members are accessed in the derived class.

Example:

class Base { public: int publicVar; protected: int protectedVar; private: int privateVar; }; // Public inheritance class Derived1 : public Base { // publicVar remains public // protectedVar remains protected // privateVar is not accessible }; // Protected inheritance class Derived2 : protected Base { // publicVar becomes protected // protectedVar remains protected // privateVar is not accessible }; // Private inheritance class Derived3 : private Base { // publicVar becomes private // protectedVar becomes private // privateVar is not accessible };

Access Rules:

  • public inheritance: public → public, protected → protected
  • protected inheritance: public → protected, protected → protected
  • private inheritance: public → private, protected → private
  • private members are never accessible in derived classes

C++ Polymorphism

Basic Polymorphism

Polymorphism means "many forms". In C++, it allows us to perform a single action in different ways. There are two types of polymorphism:

  • Compile-time polymorphism (Method Overloading)
  • Runtime polymorphism (Method Overriding)

Method Overloading Example:

class Calculator { public: // Method overloading - same name, different parameters int add(int a, int b) { return a + b; } double add(double a, double b) { return a + b; } int add(int a, int b, int c) { return a + b + c; } }; int main() { Calculator calc; cout << calc.add(5, 3); // Calls int version cout << calc.add(3.2, 4.1); // Calls double version cout << calc.add(1, 2, 3); // Calls three parameter version return 0; }

Method Overriding Example:

class Animal { public: void makeSound() { cout << "Some sound" << endl; } }; class Dog : public Animal { public: void makeSound() { // Overriding base class method cout << "Woof!" << endl; } }; class Cat : public Animal { public: void makeSound() { // Overriding base class method cout << "Meow!" << endl; } };

Virtual Functions

Virtual functions enable runtime polymorphism through a process called dynamic dispatch. When you declare a function as virtual in a base class, you can override it in any derived class.

Basic Virtual Function:

class Shape { public: virtual double getArea() { return 0; } // Virtual destructor is important when using virtual functions virtual ~Shape() {} }; class Circle : public Shape { private: double radius; public: Circle(double r) : radius(r) {} double getArea() override { // override keyword is optional but recommended return 3.14159 * radius * radius; } }; class Rectangle : public Shape { private: double width; double height; public: Rectangle(double w, double h) : width(w), height(h) {} double getArea() override { return width * height; } };

Using Virtual Functions with Pointers:

int main() { // Array of pointers to base class Shape* shapes[3]; // Creating objects of derived classes shapes[0] = new Circle(5); shapes[1] = new Rectangle(4, 6); shapes[2] = new Circle(3); // Polymorphic behavior for(int i = 0; i < 3; i++) { cout << "Area: " << shapes[i]->getArea() << endl; } // Clean up for(int i = 0; i < 3; i++) { delete shapes[i]; } return 0; }

Pure Virtual Functions:

class AbstractShape { public: // Pure virtual function makes this an abstract class virtual double getArea() = 0; virtual ~AbstractShape() {} }; class Square : public AbstractShape { private: double side; public: Square(double s) : side(s) {} // Must implement pure virtual function double getArea() override { return side * side; } };

C++ Templates

Templates allow you to write code that can work with different data types without having to rewrite the code for each type.

Example:

template class Box { private: T content; public: void setContent(T item) { content = item; } T getContent() { return content; } }; int main() { Box intBox; // Box for integers Box stringBox; // Box for strings intBox.setContent(42); stringBox.setContent("Hello"); cout << intBox.getContent() << endl; // 42 cout << stringBox.getContent() << endl; // Hello return 0; }

C++ Files

C++ provides classes for working with files: ofstream (writing), ifstream (reading), and fstream (both).

Example:

#include class FileHandler { private: string filename; public: FileHandler(string fname) : filename(fname) {} void writeToFile(string text) { ofstream file(filename); if (file.is_open()) { file << text; file.close(); } } string readFromFile() { string content; ifstream file(filename); if (file.is_open()) { getline(file, content); file.close(); } return content; } };
