Java Object-Oriented Programming (OOP)

OOP is a programming paradigm that organizes software design around data, or objects, rather than functions and logic.

Core OOP Concepts

The Four Pillars of OOP

Concept Description Example
Encapsulation Bundling data and methods that work on that data within one unit Private variables with public getters/setters
Inheritance Creating new classes that are built upon existing classes Student class inheriting from Person class
Polymorphism Objects can take on more than one form depending on the context Method overloading and overriding
Abstraction Hiding complex implementation details and showing only necessary features Abstract classes and interfaces

Benefits of OOP

  • Better code organization and modularity
  • Code reusability through inheritance
  • Flexibility through polymorphism
  • Better security through encapsulation
  • Easier maintenance and modification

Java Class Attributes

Class attributes are variables within a class that describe the characteristics of objects.

Types of Attributes

Access Modifiers

Modifier Description Accessibility
private Only accessible within the class Class only
public Accessible from any other class Any class
protected Accessible in same package and subclasses Package + Subclasses
(default) Only accessible in same package Package only

Working with Attributes

public class Person { // Private attributes with different data types private String name; private int age; private double height; private boolean isStudent; // Static attribute (shared by all instances) public static int personCount = 0; // Final attribute (constant) public final String SPECIES = "Homo Sapiens"; // Getter methods public String getName() { return name; } public int getAge() { return age; } // Setter methods public void setName(String name) { this.name = name; } public void setAge(int age) { if (age > 0) { this.age = age; } } }

Best Practices

  • Use private attributes to implement encapsulation
  • Provide public getter and setter methods
  • Validate data in setter methods
  • Use meaningful names for attributes
  • Initialize attributes with default values when appropriate

Method Overloading

public class Shape { // Method overloading example public double calculateArea(double radius) { // Circle area return Math.PI * radius * radius; } public double calculateArea(double length, double width) { // Rectangle area return length * width; } public double calculateArea(double a, double b, double c) { // Triangle area using Heron's formula double s = (a + b + c) / 2; return Math.sqrt(s * (s - a) * (s - b) * (s - c)); } }

Java Constructors

A constructor is a special method used to initialize objects. It is called when an object is created using the new keyword.

Types of Constructors

Constructor Categories

  • Default Constructor (No parameters)
  • Parameterized Constructor
  • Copy Constructor
  • Chained Constructors (Constructor Overloading)

Constructor Implementation

public class Student { private String name; private int age; private double gpa; // Default constructor public Student() { this.name = "Unknown"; this.age = 0; this.gpa = 0.0; } // Parameterized constructor public Student(String name, int age) { this.name = name; this.age = age; this.gpa = 0.0; } // Constructor with all parameters public Student(String name, int age, double gpa) { this.name = name; this.age = age; this.gpa = gpa; } // Copy constructor public Student(Student other) { this.name = other.name; this.age = other.age; this.gpa = other.gpa; } }

Constructor Chaining

public class Person { private String name; private int age; private String address; private String phone; // Constructor with all parameters public Person(String name, int age, String address, String phone) { this.name = name; this.age = age; this.address = address; this.phone = phone; } // Constructor with three parameters public Person(String name, int age, String address) { this(name, age, address, "Unknown"); // Calls four-parameter constructor } // Constructor with two parameters public Person(String name, int age) { this(name, age, "Unknown"); // Calls three-parameter constructor } // Constructor with one parameter public Person(String name) { this(name, 0); // Calls two-parameter constructor } // Default constructor public Person() { this("Unknown"); // Calls one-parameter constructor } }

Java Encapsulation

Encapsulation is the mechanism of wrapping data and code together as a single unit and hiding internal details from the outside world.

Benefits of Encapsulation

  • Better control of class attributes and methods
  • Class attributes can be made read-only or write-only
  • Increased security of data
  • Flexibility to change code implementation

Implementing Encapsulation

public class BankAccount { // Private attributes private String accountNumber; private double balance; private String accountHolder; // Constructor public BankAccount(String accountHolder) { this.accountHolder = accountHolder; this.accountNumber = generateAccountNumber(); this.balance = 0.0; } // Private method private String generateAccountNumber() { return "ACC" + System.currentTimeMillis(); } // Public getter methods public String getAccountNumber() { return accountNumber; } public double getBalance() { return balance; } public String getAccountHolder() { return accountHolder; } // Public setter method with validation public void setAccountHolder(String accountHolder) { if (accountHolder != null && !accountHolder.trim().isEmpty()) { this.accountHolder = accountHolder; } } // Public methods for transactions public void deposit(double amount) { if (amount > 0) { balance += amount; } } public boolean withdraw(double amount) { if (amount > 0 && amount <= balance) { balance -= amount; return true; } return false; } }

public class Student { // Private fields private String studentId; private String name; private int age; private double[] grades; // Constructor public Student(String name, int age) { this.studentId = generateId(); this.name = name; this.age = age; this.grades = new double[0]; } // Private helper method private String generateId() { return "STU" + System.currentTimeMillis(); } // Getter methods public String getStudentId() { return studentId; } public String getName() { return name; } public int getAge() { return age; } public double[] getGrades() { return grades.clone(); // Return copy to maintain encapsulation } // Setter methods with validation public void setName(String name) { if (name != null && !name.trim().isEmpty()) { this.name = name; } } public void setAge(int age) { if (age > 0 && age < 120) { this.age = age; } } // Method to add grade public void addGrade(double grade) { if (grade >= 0 && grade <= 100) { double[] newGrades = new double[grades.length + 1]; System.arraycopy(grades, 0, newGrades, 0, grades.length); newGrades[grades.length] = grade; grades = newGrades; } } // Method to calculate average public double calculateAverage() { if (grades.length == 0) return 0; double sum = 0; for (double grade : grades) { sum += grade; } return sum / grades.length; } }

Java Inheritance

Inheritance is a mechanism that allows a class to inherit attributes and methods from another class. It provides code reusability and establishes a relationship between parent and child classes.

Types of Inheritance

Inheritance Relationships

  • Single Inheritance
  • Multilevel Inheritance
  • Hierarchical Inheritance
  • Multiple Inheritance (through interfaces)

Implementing Inheritance

// Parent class public class Animal { protected String name; protected String species; public Animal(String name, String species) { this.name = name; this.species = species; } public void eat() { System.out.println(name + " is eating"); } public void sleep() { System.out.println(name + " is sleeping"); } } // Child class public class Dog extends Animal { private String breed; public Dog(String name, String breed) { super(name, "Canis lupus familiaris"); this.breed = breed; } public void bark() { System.out.println(name + " is barking"); } // Method overriding @Override public void eat() { System.out.println(name + " the " + breed + " is eating dog food"); } } // Another child class public class Cat extends Animal { private boolean isIndoor; public Cat(String name, boolean isIndoor) { super(name, "Felis catus"); this.isIndoor = isIndoor; } public void meow() { System.out.println(name + " is meowing"); } @Override public void sleep() { if (isIndoor) { System.out.println(name + " is sleeping on the couch"); } else { System.out.println(name + " is sleeping outside"); } } }

Java Polymorphism

Polymorphism means "many forms", and it occurs when we have many classes that are related to each other by inheritance.

Types of Polymorphism

1. Method Overriding (Runtime Polymorphism)

// Parent class class Animal { public void makeSound() { System.out.println("Some sound"); } } // Child classes class Dog extends Animal { @Override public void makeSound() { System.out.println("Woof!"); } } class Cat extends Animal { @Override public void makeSound() { System.out.println("Meow!"); } } // Using polymorphism public class Main { public static void main(String[] args) { Animal myDog = new Dog(); // Animal reference, Dog object Animal myCat = new Cat(); // Animal reference, Cat object myDog.makeSound(); // Outputs: Woof! myCat.makeSound(); // Outputs: Meow! } }

2. Method Overloading (Compile-time Polymorphism)

class Calculator { // Method overloading public int add(int a, int b) { return a + b; } public double add(double a, double b) { return a + b; } public int add(int a, int b, int c) { return a + b + c; } } public class Main { public static void main(String[] args) { Calculator calc = new Calculator(); System.out.println(calc.add(5, 3)); // calls first method System.out.println(calc.add(4.5, 3.2)); // calls second method System.out.println(calc.add(2, 3, 4)); // calls third method } }

Java Inner Classes

In Java, it is possible to nest classes (a class within a class). The purpose of nested classes is to group classes that belong together and to achieve encapsulation.

Types of Inner Classes

1. Non-Static Inner Class

class OuterClass { private int x = 10; class InnerClass { public void display() { System.out.println("Value of x: " + x); // Can access outer class members } } public void createInner() { InnerClass inner = new InnerClass(); inner.display(); } } public class Main { public static void main(String[] args) { OuterClass outer = new OuterClass(); OuterClass.InnerClass inner = outer.new InnerClass(); inner.display(); } }

2. Static Inner Class

class OuterClass { private static int x = 10; static class StaticInnerClass { public void display() { System.out.println("Static x: " + x); // Can only access static members } } } public class Main { public static void main(String[] args) { OuterClass.StaticInnerClass inner = new OuterClass.StaticInnerClass(); inner.display(); } }

3. Local Inner Class

public class OuterClass { public void method() { class LocalInnerClass { public void print() { System.out.println("Local inner class method"); } } LocalInnerClass local = new LocalInnerClass(); local.print(); } }

4. Anonymous Inner Class

interface Message { void display(); } public class Main { public static void main(String[] args) { Message message = new Message() { @Override public void display() { System.out.println("Anonymous inner class"); } }; message.display(); } }

Java Abstraction

Abstraction is a process of hiding implementation details and showing only functionality to the user. It can be achieved with abstract classes and interfaces.

Abstract Classes

Abstract Class Example

abstract class Animal { protected String name; public Animal(String name) { this.name = name; } // Abstract method - must be implemented by subclasses abstract void makeSound(); // Concrete method public void sleep() { System.out.println(name + " is sleeping"); } } class Dog extends Animal { public Dog(String name) { super(name); } @Override void makeSound() { System.out.println(name + " says: Woof!"); } } class Cat extends Animal { public Cat(String name) { super(name); } @Override void makeSound() { System.out.println(name + " says: Meow!"); } }

Interfaces

Interface Example

interface Vehicle { void start(); void stop(); default void horn() { // Default method System.out.println("Beep!"); } } class Car implements Vehicle { @Override public void start() { System.out.println("Car is starting"); } @Override public void stop() { System.out.println("Car is stopping"); } } class Motorcycle implements Vehicle { @Override public void start() { System.out.println("Motorcycle is starting"); } @Override public void stop() { System.out.println("Motorcycle is stopping"); } @Override public void horn() { System.out.println("Motorcycle horn!"); } }

Java Interface

An interface is a completely abstract class that contains only abstract methods. It can be used to achieve abstraction and multiple inheritance in Java.

Interface Basics

Basic Interface Example

interface Animal { void makeSound(); // abstract by default void eat(); // abstract by default } class Cat implements Animal { @Override public void makeSound() { System.out.println("Meow!"); } @Override public void eat() { System.out.println("Cat is eating fish"); } } class Dog implements Animal { @Override public void makeSound() { System.out.println("Woof!"); } @Override public void eat() { System.out.println("Dog is eating bone"); } }

Multiple Interfaces

Implementing Multiple Interfaces

interface Flyable { void fly(); } interface Swimmable { void swim(); } class Duck implements Animal, Flyable, Swimmable { @Override public void makeSound() { System.out.println("Quack!"); } @Override public void eat() { System.out.println("Duck is eating seeds"); } @Override public void fly() { System.out.println("Duck is flying"); } @Override public void swim() { System.out.println("Duck is swimming"); } }
